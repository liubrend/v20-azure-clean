# Implementation tasks

One file per unit-sized piece of work, small enough for one PR. Number them
`task-001.md`, `task-002.md`, … Copy `task.md` to start.

Each task traces back to a spec scenario in `docs/specs/` and/or a decision in
`docs/adr/` — it breaks that spec/ADR down into the concrete files an
implementer (human or AI coding agent) touches, in the order to touch them, so
"understand the spec" and "write the code" are separate steps. If a task needs
"and" to describe its goal, split it into two tasks.
